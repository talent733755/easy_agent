# 知识库与网络搜索决策规范

**日期:** 2026-05-08
**状态:** 已批准

---

## 背景

当前 Agent 可同时访问知识库（RAG 检索）和网络搜索（web_search 工具），但缺乏决策规范，导致可能在有知识库结果时仍调用搜索，或在知识库缺失时给出不可靠的回答。

---

## 决策规范

### 决策树

```
用户提问
    │
    ▼
知识库有结果?
    │
    ├── 是 → 基于知识库回答，标注【来源：文件名】
    │
    └── 否 → 是美容行业问题?
                │
                ├── 是 → 回复：「该问题超出知识库范围，建议咨询门店顾问」
                │
                └── 否 → 可使用 web_search 补充
                            │
                            ├── 有结果 → 综合回答，标注搜索来源
                            │
                            └── 无结果 → 回复：「抱歉，当前无法回答该问题，建议联系工作人员」
```

### 工具使用规则

| 场景 | 知识库 | web_search |
|------|--------|-----------|
| 知识库有结果 | ✅ 必须用 | ❌ 禁止用 |
| 知识库无结果 + 美容行业问题 | — | ❌ 禁止用 |
| 知识库无结果 + 通用/实时问题 | — | ✅ 可用 |
| 搜索也无结果 | — | — → 诚实回复 |

### 回答格式要求

- 使用知识库内容时，**必须**在回答末尾标注来源
  - 格式：`【来源：products/嫩小白.txt】`
  - 多来源：`【来源：products/嫩小白.txt、procedures/客户接待流程.txt】`
- 搜索结果作为补充时，标注搜索来源（URL 标题）

---

## 实现方式

仅修改 `src/nodes/agent_node.py` 中的 `SYSTEM_PROMPT`，在 system prompt 中向 Agent 说明决策规范，无需新增节点或改流程。

---

## 验收标准

1. 问"敏感肌适合什么产品" → 返回知识库结果，标注来源
2. 问"你们店明天有什么优惠" → 回复超范围提示
3. 问"今天北京天气如何" → 允许搜索网络
4. 问"娇莉芙的肌源新生项目是什么" → 返回知识库结果，标注来源
